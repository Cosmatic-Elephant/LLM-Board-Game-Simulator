## 세션 10 (완료) — 멀티플레이 핵심 루프 구현

- Socket.io 방 생성/참가/퇴장, 색상·AI 모델·게임 설정 실시간 동기화 (`server.ts`, `src/app/page.tsx`)

- LLM/깡통은 전부 무작위 자동 진행(실제 LLM API 미연동), `/multi` UI는 정적 렌더링만(연출 없음), 이탈/타임아웃 처리 없음

## 세션 11 (완료, 2026-07-10~11) — 멀티플레이 완성도 작업

- **이탈/종료 처리**: 게임 중 호스트 이탈 시 방 삭제 + `host-left`, 게스트 이탈 시 해당 슬롯 깡통 대체 + `player-left`, 호스트의 명시적 "게임 종료"는 `end-game`/`game-ended`로 전원 퇴장. 우측 하단 나가기 버튼(호스트/게스트 문구 분기, 기존 `leave-room`/`end-game` 재사용)
- **턴 타임아웃**: 사람 플레이어 15초 미행동 시 서버가 깡통처럼 자동 행동, 5초 이하 카운트다운 UI + 굴리기 버튼/카지노 테두리 점멸(본인 클라이언트만)
- **실제 LLM API 연동**: 서버의 자동 베팅 로직을 `getLLMAction()` 호출로 교체(단일 액션 최적화·깡통 랜덤 폴백 유지, 실패 시 랜덤 대체)
- **베팅 말풍선**: 사람/봇/타임아웃 구분 + LLM reasoning 포함, 서버가 `bet-bubble`로 구조화된 정보 전송
- **턴 연출 이식**(싱글 → 멀티): 주사위 그룹화, 프리롤 페이드인, 굴리기 셔플, 베팅 퇴장 애니메이션 — 서버 브로드캐스트를 지연 반영하는 상태 큐(`stateQueueRef`/`processQueue`) 아키텍처로 구현
- **호버 하이라이팅**: 카지노↔주사위 상호 하이라이팅(클라이언트 전용), 타임아웃 점멸보다 호버가 우선 표시
- **정산 연출 이식**(싱글 → 멀티): 서버가 정산 전 카지노 스냅샷 + 단계별 이벤트(`scoring-steps`)를 먼저 보내고 round-end/game-end `game-state`가 뒤따름, 클라이언트가 싱글과 동일한 순서로 재생(동률 배제 → 순위 하이라이트 → 지폐 페이드+소지금 증가 → 테이블 클리어), 스킵 버튼(호스트 전용, `skip-scoring`/`scoring-skipped`)
- **버그 수정**: 멀티 종료 후 싱글 로비 설정이 남은 sessionStorage 값에 가려지던 문제(죽은 코드 제거), 베팅 말풍선이 연출 시작 전에 너무 일찍 뜨던 문제(큐로 지연)

## 세션 12 (완료, 2026-07-14) — 정산 연출 마무리 · 싱글 이식 · NVIDIA NIM 지원

- **정산 진입 흐름 개선(멀티)**: 전원 주사위 소진 후 즉시 정산 연출로 넘어가지 않고 1.5초 대기 + 중앙 안내 텍스트("라운드 종료 / 정산을 시작합니다.") 표시, 대기 종료 시 남아있는 말풍선을 hover 상태와 무관하게 일괄 제거한 뒤 정산 연출 시작. "정산 중" 라벨을 스킵 버튼 위에 배치, 스킵 버튼은 실제 연출 진행 중에만 노출. `+금액` 텍스트 크기 확대(`text-sm` → `text-xl`)
- **게임 종료 화면 신설(멀티)**: 라운드 종료 화면의 "게임 종료" 클릭 시(서버에 알리지 않는 로컬 전용 전환) 우승자 문구(`{이름}가 {금액}원으로 우승!`, 동점자 전원 표시) + 나가기/다시하기 버튼 노출. "다시하기"는 호스트 전용이며 새 `restart-game` 소켓 이벤트로 동일 플레이어 구성(색상·isLLM·이름·모델)을 그대로 재사용해 소지금·지폐·순서를 초기화한 게임을 방 전체에 브로드캐스트(`server.ts`: `buildInitialGameState`를 `buildGameStateFromConfigs`로 공용화)
- **버그 수정(멀티)**: 정산 진입 시 화면이 다른 `<main>` 분기로 교체되면서 마지막 말풍선이 리마운트되어 진입 애니메이션이 다시 재생되던 문제(`bubbleEnteredKeyRef`로 이미 재생된 말풍선의 key를 색상별로 기억), 같은 원인으로 "차례" 표시(위로 이동)가 다음 화면까지 남아있던 플레이어 패널 위치 문제(화면별 `<main>`에 고유 `key`를 부여해 전환 시 항상 완전히 새로 마운트)
- **싱글플레이 정산 연출 이식**(멀티 → 싱글, `game/page.tsx`): 정산 대기 1.5초+안내 텍스트, "정산 중" 라벨 위치, `+금액` 텍스트 크기, 라운드 종료/게임 종료 화면을 카지노 보드 없이 플레이어 패널이 중앙에 오는 별도 `<main>` 레이아웃으로 분리(`renderPlayerPanel`/`renderExitControls` 헬퍼로 3개 화면 공용). 게임 종료 화면 문구·버튼도 멀티와 동일하게 통일(나가기/다시하기, 우승자 문구 포맷). 멀티의 큐/소켓 관련 로직(`processQueue`, `stateQueueRef` 등)은 이식 대상에서 제외 — 싱글은 로컬 상태만으로 충분
- **NVIDIA NIM 프로바이더 추가**: `llm-client.ts`에 모델 ID가 `nvapi-` 접두사거나 `/`를 포함하면(예: `meta/llama-3.1-70b-instruct`) NVIDIA로 라우팅하는 분기 추가, 기존 `openai` SDK를 그대로 재사용하되 `baseURL: https://integrate.api.nvidia.com/v1` + `NVIDIA_API_KEY`로 설정. `.env.local.example`에 키 항목 추가. 로비 모델 콤보박스에 Llama 3.1 70B(`meta/llama-3.1-70b-instruct`) / GLM-4 (무료)(`zhipuai/glm-4`) 추가하면서 `MODEL_OPTIONS`를 문자열 배열 → `{ value, label }` 구조로 변경(모델 ID와 표시명이 달라야 하므로)
- **정리**: 싱글의 Q키 테스트 단축키(`// TEST ONLY` 주석 위치, `qHandlerRef` + `keydown` 리스너) 완전 제거

## 세션 13 (완료, 2026-07-15) — 게임 선택 UI · 컴포넌트 분리 · 문서 재배치

- **게임 선택 기능 추가** (`src/app/page.tsx`): 로비에 `GAME_OPTIONS` 상수(라스베가스/요트 다이스) 도입. "게임 변경" 버튼 클릭 시 외부 클릭으로 닫히는 리스트박스가 열리고, 목록 항목 클릭 시 선택 게임 전환 + 현재 선택 항목 강조 표시. 선택된 게임에 따라 중앙 타이틀/서브타이틀이 교체되며, 싱글플레이 버튼은 라스베가스일 때만 기존 설정 팝업으로 연결되고(요트 다이스는 `console.log` 플레이스홀더), 멀티플레이 버튼은 라스베가스에서만 활성화(요트 다이스는 `disabled` + `cursor-not-allowed`)
- **리스트박스 다듬기**: 목록 마지막에 클릭 불가능한 "곧 새로운 게임이 추가됩니다..." 안내 항목 추가(흐린 색상으로 구분), 목록에 `max-h-[300px]` + 세로 스크롤 적용
- **컴포넌트 분리** (`page.tsx` 946줄 → 65줄, 커밋 `37a1f89`): 게임 무관 UI 프리미티브(`Toggle`/`SelectField`/`ReadOnlyBox`/`PopupHeader`/`GameSelectListbox`)를 `src/components/ui/`로, 라스베가스 전용 설정 팝업(`PlayerSetupPopup`/`MultiplayerPopup`/`MultiplayerRoomPopup`)과 색상 종속 컴포넌트(`ColorSelect`/`ColorDisplay`/`AIBadge`) · 상수(`constants.ts`)를 `src/components/las-vegas/`로 이동. `ColorSelect`와 `GameSelectListbox`에 중복돼 있던 "외부 클릭 시 닫힘" 로직은 `src/hooks/useClickOutside.ts` 훅으로 추출해 교체. 순수 파일 이동·import 정리만 수행했으며 게임 로직·동작은 변경하지 않음
- **DESIGN.md 재배치**: 루트의 `DESIGN.md`를 `docs/las-vegas/DESIGN.md`로 이동(`git mv`로 이력 보존), `CLAUDE.md`(참조 3곳)와 `src/lib/bill-setup.ts`/`game-engine.ts`/`llm-client.ts`/`src/types/game.ts` 주석의 경로를 함께 갱신. `HISTORY.md`는 과거 시점을 있는 그대로 기록한 아카이브라 판단해 의도적으로 미수정
- 각 단계마다 `npx tsc --noEmit` 통과 확인

## 현재 상태

싱글·멀티 모두 정산 연출(대기 → 애니메이션 → 라운드 종료/게임 종료 화면)이 동일한 수준으로 동작한다. 멀티는 방 생성부터 게임 종료·다시하기까지 엔드투엔드로 동작하며, LLM 프로바이더로 Anthropic/OpenAI/Google/NVIDIA NIM 4종을 지원한다(NVIDIA는 타입체크만 확인했고 실제 API 키로 호출 검증은 아직 안 함). 로비에 게임 선택 UI가 추가되어 라스베가스/요트 다이스 중 고를 수 있으나, 실제 플레이 가능한 것은 라스베가스뿐이다(요트 다이스는 선택·타이틀 표시만 되고 설정 팝업은 플레이스홀더). `npx tsc --noEmit` 에러 없음.
